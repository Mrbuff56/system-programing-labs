Задание 3-4: Диагностика сетевых соединений

Использованные утилиты и их назначение:

 `curl -I`  Получение HTTP заголовков
 `curl -v`  Детальный HTTP запрос 
 `ping`  Проверка доступности 
 `ss` Просмотр сокетов
 `netstat`  Сетевые соединения 
 `lsof`  Файлы и сокеты процесса 
 `nc`  Низкоуровневый TCP

 Наблюдения:
1. HTTP-заголовки (curl -I):
- **Server**: SimpleHTTP/0.6 Python/3.12.3 - простой Python сервер
- **Content-Type**: text/html - статические HTML файлы
- **Content-Length**: 3661 байт - размер ответа (директория /home/dafna)

2. Детали соединения (curl -v):
- IPv6 connection refused → используется IPv4 (127.0.0.1)
- HTTP 1.0 (сервер не поддерживает keep-alive)
- Заголовки запроса: User-Agent, Accept

3. Состояние порта (ss, netstat):
- **Состояние**: LISTEN (ожидание подключений)
- **Адрес**: 0.0.0.0:8080 (слушает на всех интерфейсах)
- **PID/Program**: 419/python3
- **TIME_WAIT** соединения после curl запросов

4. Информация о процессе (lsof):
- **COMMAND**: python3
- **PID**: 419
- **FD**: 3u (файловый дескриптор 3 для сокета)
- **TYPE**: IPv4
- **NAME**: *:http-alt (8080)

5. Доступность (ping):
- 0% packet loss - стабильное соединение
- Время: 0.022-0.083ms - очень низкая задержка (localhost)

6. Низкоуровневый анализ (nc):
- Можно отправить сырой HTTP запрос
- Сервер отвечает полным HTTP ответом с телом
- HTML содержит директорию /home/dafna
