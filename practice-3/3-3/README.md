Задание 3-3: Сокеты TCP
Требования:
• Используйте socket(), bind(), listen(), accept(), connect(), send(), recv(), close().
• Реализуйте клиент и сервер на C.
• Обрабатывайте ошибки.
• Компиляция: gcc -Wall -Wextra -o task task.c.

Вариант 6. Сервер принимает имя файла от клиента и,
если файл существует, отправляет его содержимое.

Используемые функции:
socket() - создает сокет
bind() - привязывает сокет к IP-адресу и порту
listen() - переводит сокет в режим ожидания подключений
accept() - принимает входящее соединение от клиента
connect() -	установливает соединение с сервером
send() - отправляет данные через установленное соединение
recv() - получает данные из сокета
close() - закрывает файловый дескриптор
fork() - создает дочерний процесс
stat() - проверка существования файла и получение информации о нем
read() - чтение данныx

Алгоритм работы программы:
Инициализация функция main:
Создает дочерний процесс через fork()
Дочерний процесс запускает сервер
Родительский процесс запускает клиент
Родитель ожидает завершения дочернего процесса через wait()

Серверная часть run_server():
создает TCP-сокет
устанавливает опцию SO_REUSEADDR для переиспользования порта
привязывает сокет к порту
переводит сокет в режим прослушивания
блокируется до подключения клиента
При подключении создается новый сокет client_fd для общения с этим клиентом
Вызывается handle_client()

handle_client():
получает имя файла от клиента
проверяет существование файла
открывает файл (если существует)
Цикл: read() из файла → send() клиенту
отправляет маркер "EOF" для обозначения конца передачи
закрывает файл и сокет клиента

Клиентская часть run_client():
создает клиентский сокет socket()
преобразует IP-адрес "127.0.0.1" в бинарный формат
устанавливает соединение с сервером
читает имя файла с клавиатуры
отправляет имя файла серверу
