Задание 2-3: Обработка сигналов
Требования:
•Используйте signal() или sigaction()
•Обработчик должен быть async-signal-safe (используйте write() вместо printf())
•Флаги: volatile sig_atomic_t

Вариант 6: Родитель создаёт дочерний процесс. Дочерний засыпает на 30 секунд.
Родитель через 5 секунд отправляет SIGUSR1. Дочерний обрабатывает и выводит
"Signal received"

Используемые функции:
fork() - создание нового процесса
signal() - установка обработчика сигнала
kill() - отправка сигнала процессу
wait() - ожидание завершения дочернего процесса
sleep() - приостановка выполнения процесса
write() - асинхронно-безопасный вывод 

Создает дочерний процесс с помощью fork()
В ветке дочернего процесса:
Устанавливает обработчик сигнала
Засыпает на 30 секунд
Проверяет флаг после пробуждения
В ветке родительского процесса:
Ждет 5 секунд
Отправляет сигнал SIGUSR1
Ожидает завершения дочернего процесса
Для запуска используется ./task

Пример вывода:
komar@DESKTOP-DAHGGQB:/mnt/c/Users/komar/Downloads/system-programming-labs_Old-main/system-programming-labs/practice-2/2-3$ ./task
Дочерний процесс создан (PID: 1550)
Дочерний процесс засыпает
Родительский процесс: sending SIGUSR1...
Signal received
Дочерний процесс получил сигнал
Дочерний процесс завершился
